Die Technologien hinter der Magie

PHP

LuMAX W3B verwendet PHP5 und zukünftig PHP7 für die Erstellung von Webseiten und Applikationen. Dabei legen wir besonderen Wert auf schlanken, sauberen und effizienten Code.

Perl

Perl ist und bleibt eine der wertvollsten Paradigmen übergreifenden Programmiersprachen aller Zeiten, wir setzen diese Sprache für schnelle und Systemnahe Lösungen ein.

C / C++

Diese Sprache ist ein mächtiges und extrem schnelles Werkzeug. Wir verwenden C/C++ für ARM/ Atmel Mikrocontroller Programmierung und schnelle CGI Prozesse, die die volle Power der modernen Server gezielt nutzen.

MySQL

Dieses relationale Datenbankverwaltungssystem wird von fast allen Free und Open Source System Herstellern eingesetzt. Leider wurde MySQL aufgekauft und ist teilweise kostenpflichtig!

PostgreSQL

PostgreSQL ist ein freies objektrelationales Datenbankmanagementsystem, welches wir für LuMAX W3B eigene Software und Kundenspezifische Projekte verwendet wird.

Linux

Linux ist der Kern einer positiven und weltweiten Entwicklung an freien Betriebssystemen wie Ubuntu, Debian, Fedora oder Gentoo. LuMAX W3B macht sich für den Einsatz dieser Distributionen stark und setzt diese selbst ein!

Croogo CMS Events Management and Scalability

The Croogo CMS is based on CakePHP and use Cake's awesome Events System. My customers ask me often what happen when their needs on a Portal, Application or CMS grow. Do we build a new Website again, and again every 2-4 Years? - My answer is "No". Code, Sprache und Paradigmen

Ein Programmierstil ist ein Paradigma und umgekehrt. Anders definiert, ein Paradigma basiert auf einer Sprache, somit Ihren Prinzipien und bietet dem Programmierer Entfaltungsmöglichkeiten und eine Basis sich in einer Programmiersprache auszudrücken. Das Resultat sind Variablen, Funktionen, Objekte, Klassen, Prozesse und Applikationen, umgangssprachlich Software genannt.

Das berühmt-berüchtigte 'Hello World!' ('Hallo Welt!') und bei jedem Anfänger und alteingesessenen Programmierer bekannte erste Programm.

Hier in C:

#include <stdio.h>
int main() {
    printf("Hello World!\n");
    return 0;
}

Hier in C++:

#include <iostream>
int main() {
    std::cout << "Hello World!" << std::endl;
    return 0;
}

Hier in PHP:

<?php echo 'Hello World!'; ?>

Hier in Java:

class HelloWorld {
    public static void main (String args[]) {
        System.out.println ('Hello World!');
    }
}

Was besonders auffällt, sind die unterschiedlichen Schreibweisen in der jeweiligen Sprache. In PHP reicht die 'echo' Funktion und in Java bedarf es ein paar mehr Zeilen Code um die 'System.out.println' Funktion und somit die Ausgabe des Programms anzustoßen.

PHP ist die Global meistgenutzte Serverbasierte Interpretersprache der Welt, das heißt der Code wird zur Laufzeit durch den Server eingelesent, analysiert und ausgeführt. Diese Tatsache ist den meißen nicht Programmierern, zudenen 90% unserer Kunden und Auftraggeber zählen, schlichtweg nicht bewusst. Was einmal interpretiert wurde, ist nicht veränderbar, sondern muss neu abgerufen und wieder interpretiert werden, da es schon an den Klienten (Empfänger) gesendet wurde.
